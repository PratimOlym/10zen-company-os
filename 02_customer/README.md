# Review Process

Purpose:

Keep all customer-facing assets consistent.

Trigger:

Run review when:

* executive_deck changes
* one_pager changes
* service_portfolio changes
* new customer assets are created
* before sending assets externally

Workflow:

Update Asset

↓

Open CUSTOMER_REVIEW.md

↓

Run Consistency Checklist

↓

Record findings

↓

Apply fixes

↓

Commit changes

Outputs:

Updated assets

Review history

Improved consistency

Rule:

Do not modify customer-facing assets without updating review records.
